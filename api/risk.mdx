## Update Protocol Risk Config

Update or create risk configuration for a protocol.

```bash cURL
curl -X PUT \
  -H "x-api-key: dev-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "maxSlippageBps": 75,
    "maxPoolConcentrationBps": 4000,
    "gaslessEligible": false,
```

## Delta Guard

The delta guard system verifies expected vs observed lamport movement in transactions.

### How It Works

1. Before transaction: Record wallet balance
2. After confirmation: Check actual balance change
3. Compare to expected change from transaction intent
4. If deviation exceeds threshold: Flag or auto-pause

### Configuration

**Global Tolerance:**
```bash
DELTA_GUARD_ABSOLUTE_TOLERANCE_LAMPORTS=5000
```

**Per-Protocol Threshold:**
```json
{
  "protocol": "jupiter",
  "deltaVarianceBpsThreshold": 500
}
```

### Example

```
Expected change: -1,000,000 lamports (swap input)
Actual change:   -1,004,500 lamports (with fees)
Deviance:          4,500 lamports
Threshold:         5,000 lamports
Result: PASS
```

## Chaos Engineering

Chaos mode injects controlled failures for resilience testing.

### Get Chaos Config

```bash cURL
curl -H "x-api-key: dev-api-key" \
     http://localhost:3000/api/v1/chaos
```

### Response

<ResponseField name="enabled" type="boolean" required>
  Whether chaos mode is enabled
</ResponseField>

<ResponseField name="failureRates" type="object">
  Failure rate percentages by stage
  <Expandable title="stages">
    <ResponseField name="simulation" type="number">
      Simulation failure rate (0.0-1.0)
    </ResponseField>
    <ResponseField name="build" type="number">
      Build failure rate (0.0-1.0)
    </ResponseField>
    <ResponseField name="sign" type="number">
      Signing failure rate (0.0-1.0)
    </ResponseField>
    <ResponseField name="send" type="number">
      Send failure rate (0.0-1.0)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="latencyMs" type="number">
  Additional latency to inject (milliseconds)
</ResponseField>

```json
{
  "status": "success",
  "data": {
    "enabled": false,
    "failureRates": {
      "simulation": 0.0,
      "build": 0.0,
      "sign": 0.0,
      "send": 0.0
    },
    "latencyMs": 0
  }
}
```

### Update Chaos Config

```bash cURL
curl -X PUT \
  -H "x-api-key: dev-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "enabled": true,
    "failureRates": {
      "simulation": 0.1,
      "build": 0.05
    },
    "latencyMs": 100
  }' \
  http://localhost:3000/api/v1/chaos
```

### Request Body

<ParamField body="enabled" type="boolean">
  Enable/disable chaos mode
</ParamField>

<ParamField body="failureRates" type="object">
  Failure rates by stage (0.0-1.0)
</ParamField>

<ParamField body="latencyMs" type="number">
  Artificial latency in milliseconds
</ParamField>

<Warning>
  Only enable chaos mode in testing environments. Never use in production.
</Warning>
